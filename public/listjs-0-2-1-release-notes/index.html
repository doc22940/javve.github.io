List.js 0.2.1: Release notes

Apr 24th, 2012

Major changes

  • A Fuzzy Search plugin is added Thanks Luuk van Egeraat for this commit
  • The filter function in list.filter(filterFunction) now takes an Item instead of values which results in greater possibilities, e.g. by using item.elm. Thanks Bob Spryn

Other changes

  • Change plugin functionality so that plugins are created with `Plugin.call(list, locals, options)?, so that plugins extends the List-objects that they are created in.
  • Add possibility to initiate with a element instead of a ID.

Bug fixes

  • Don’t break .hasClass() if a element doesn’t have a class. Thanks Jonas Forsberg
  • Do not require items to contain all valueNames on initiation.

Go to Listjs.com and download tha latest version!
